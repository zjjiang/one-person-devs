## 🐛 问题修复

### fix: 修复 CapabilityRegistry 实例共享问题

问题：
- with_project_overrides() 直接赋值 _external_providers
- 导致多个项目共享同一个 dict 引用
- 理论上可能导致状态泄露
解决方案：
- 深拷贝 _external_providers 字典
- 确保每个项目的 registry 完全独立
- 添加注释说明 _capabilities 浅拷贝的安全性

*提交: `a46c2d4` - *

### fix: 修复高优先级并发安全问题

1. GitHubProvider 阻塞调用
   - 将所有 subprocess.run 改为 asyncio.create_subprocess_exec
   - 避免阻塞 asyncio 事件循环
   - 提升并发性能
2. _clone_tasks 竞态条件
   - 添加 _clone_locks 字典保护并发访问
   - 使用 asyncio.Lock 确保同一项目只启动一个 clone 任务
   - 防止资源浪费
3. _site_url 缓存竞态条件
   - 添加 _site_url_lock 保护初始化
   - 使用 double-check locking 模式
   - 改为 async 函数，所有调用点添加 await

*提交: `40fde12` - *

### fix: 修复 ClaudeCodeProvider 的并发安全问题

问题：
- _apply_env() 直接修改全局 os.environ，存在竞态条件
- 多个项目并发调用时会互相覆盖环境变量
- 导致使用错误的 auth_token 或 base_url
解决方案：
- 使用 ClaudeCodeOptions 的 env 参数直接传递环境变量
- 移除 _apply_env() 和 _restore_env() 方法
- 确保每个请求的配置完全隔离
影响：
- 修复了多项目并发执行时的认证错误
- 提升了系统的并发安全性和可靠性

*提交: `8057c91` - *
